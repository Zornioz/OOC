Kapitel 2: OSI-Schichtenmodell

Das OSI-Schichtenmodell ist ein konzeptionelles Framework, das die Kommunikation in Netzwerken in sieben Schichten unterteilt. Es hilft, die verschiedenen Funktionen der Netzwerktechnologie zu verstehen.

Die 7 Schichten

  1. Physical Layer (Physikalische Schicht)

    Über das Kabel, das ihr in der Firma gesehen habt, werden die Daten binär übertragen. Dies bedeutet, dass elektrische Signale (bei Kupferkabeln) oder Lichtimpulse (bei Glasfaserkabeln) genutzt werden, um die Daten als Einsen und Nullen zu übertragen.

  2. Data Link Layer (Datenlink-Schicht)

    Stellt sicher, dass die Daten fehlerfrei von einem Gerät zum nächsten übertragen werden. Hierbei werden MAC-Adressen verwendet, um Geräte innerhalb eines lokalen Netzwerks zu identifizieren.

  3. Network Layer (Netzwerkschicht)

    Diese Schicht ist für die logische Adressierung und Routing zuständig. Hier kommen IP-Adressen ins Spiel, die verwendet werden, um Geräte im Internet zu identifizieren.

  4. Transport Layer (Transportschicht)

    Diese Schicht kümmert sich um die zuverlässige Übertragung von Daten zwischen Endpunkten. Hier werden Protokolle wie TCP und UDP verwendet.

  5. Session Layer (Sitzungsschicht)

    Diese Schicht verwaltet die Kommunikation zwischen Anwendungen. Hier werden Session-IDs verwendet, um die Identität einer Sitzung zu verfolgen.

  6. Presentation Layer (Darstellungsschicht)

    Hier wird sichergestellt, dass die Daten in einem verständlichen Format präsentiert werden. Diese Schicht kümmert sich um Datenformatierungen wie JSON, XML oder HTML.

  7. Application Layer (Anwendungsschicht)

    Diese oberste Schicht ermöglicht den Zugriff auf Netzwerkanwendungen. Hier arbeiten Protokolle wie HTTP, FTP und SMTP.
